# English mini-glossary for ANT

Use this when explaining technical terms to non-technical builders.

| Term | Plain-language explanation |
|---|---|
| API | A way for one program to ask another program for data or an action. |
| Endpoint | A specific address or function where an app sends or receives data. |
| Deploy | Publish the app or site so other people can use it. |
| Build | Prepare the app so it can run or be published. |
| Dependency | A library the project needs to work. |
| Package | A ready-made piece of code the project uses as a tool. |
| Repository / repo | The project folder plus its change history, often stored on GitHub. |
| Branch | A separate line of changes so the main version is not broken. |
| Commit | A saved step in the project’s change history. |
| Pull request / PR | A request to merge changes into the main version of the project. |
| Merge | Combine changes from one branch into another. |
| Environment variable | A hidden setting, often used for passwords, API keys, or service addresses. |
| Token | A digital key/pass used to prove access. |
| Schema | The shape and rules of the data. |
| Migration | A controlled change to the database structure. |
| Middleware | A layer that checks or changes a request before the main app handles it. |
| Runtime | The environment that runs the code. |
| CI/CD | Automatic checks and publishing after code changes. |
| Production | The live version used by real people. |
| Staging | A test version that behaves like production but is not for real users. |
| Authentication | Checking who the user is. |
| Authorization | Checking what the user is allowed to do. |
| Cache | Temporarily saved data used to respond faster or reduce repeated work. |
| Webhook | An automatic message one service sends another service when something happens. |
| SDK | A toolkit for working with a service or platform. |
| CLI | Controlling a tool by typing commands in the terminal. |
