# Product brief: ANT

## Одна фраза

ANT — это open-source skill pack, который учит AI-агентов объяснять техническую работу человеческим языком для non-technical builders.

## Проблема

AI-агенты умеют писать код, чинить ошибки и разворачивать проекты, но часто общаются как инженер с инженером: dependency, endpoint, middleware, migration, runtime, build, deploy, auth, token, schema. Для людей без технического бэкграунда это создаёт ощущение, что проект строится “где-то в тумане”, а пользователь не понимает, что происходит и какие есть риски.

## Решение

ANT добавляет слой понятной коммуникации:

- объяснить смысл;
- показать действие;
- сказать, что нужно сделать пользователю;
- объяснить, как проверить результат;
- назвать риски;
- вынести технику отдельно.

## Целевая аудитория

Главная аудитория:

- non-technical founders;
- solo builders;
- продакты;
- дизайнеры;
- маркетологи;
- студенты;
- creators;
- начинающие разработчики.

Не стоит позиционировать как “для девушек”. Лучше: **для людей, которые строят с AI, но не обязаны говорить на языке разработчиков**.

## Название

Бренд: **ANT** / “муравей”.

Техническое имя пакета: `ant-plain-language`.

Варианты расшифровки:

- Accessible Non-Technical Translator;
- AI Non-Technical Translator;
- Agent Non-Technical Translator.

Лучший вариант для публичного запуска: **Accessible Non-Technical Translator**, потому что он звучит шире и не ограничивает проект только AI.

## Обещание продукта

> ANT helps AI agents build with you, not talk over you.

По-русски:

> ANT помогает AI-агентам строить вместе с человеком, а не говорить поверх него.

## Почему это может стать популярным

1. Vibe coding растёт, и всё больше людей строят проекты без классического инженерного бэкграунда.
2. AI-агенты часто дают слишком технические ответы.
3. Пользователи хотят понимать, что происходит с их проектом, особенно если есть риски, деньги, production, данные или безопасность.
4. Формат skills уже переносимый: один хороший `SKILL.md` можно адаптировать под разные агенты.
5. Проект легко локализовать на разные языки.

## MVP

MVP должен включать:

- один основной `SKILL.md`;
- Claude Output Style;
- Cursor rule;
- AGENTS.md snippet;
- русский и английский README;
- 10 before/after examples;
- 10 ручных evals;
- понятный install guide.

Этот пакет уже покрывает MVP.
