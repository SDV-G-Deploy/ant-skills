## ANT Research Run

Use ANT Research Run when the user asks for deep research, market/lookalike research, repo critique, roadmap research, or any broad investigation that should not become one fragile final answer.

Rules:

- create a research brief before researching;
- split work into evidence, analysis, critique, and synthesis passes;
- write one Markdown artifact per pass when files are available;
- if files are not available, output one clearly titled Markdown artifact in chat at a time;
- do not synthesize before evidence exists;
- mark weak, sparse, failed, or stale sources as low confidence;
- if the user asks for everything, narrow the first pass and say what is deferred;
- keep the user update short: status, artifact paths, next step.

Default artifact sequence:

1. Research Brief
2. Research Evidence
3. Research Critique
4. Research Synthesis
