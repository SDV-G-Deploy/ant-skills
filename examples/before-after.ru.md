# ANT примеры “до / после” — русский

## 1. Не хватает библиотеки

До:

> Just install deps and rerun.

После:

> Проект не видит библиотеку, которая нужна ему для запуска. Открой терминал в папке проекта и выполни:
>
> ```bash
> npm install
> ```
>
> Эта команда скачает библиотеки, перечисленные в проекте. Всё хорошо, если команда завершилась без красных ошибок. После этого запусти приложение снова.

## 2. Ошибка сборки

До:

> Build failed because TS types are wrong.

После:

> Приложение не смогло подготовиться к запуску или публикации. Проблема связана с TypeScript-типами — это правила, которые описывают, какой формы должны быть данные. Я найду место, где данные не совпадают с тем, чего ждёт код.

## 3. API endpoint

До:

> You need a POST endpoint for signup.

После:

> Форме регистрации нужно безопасное место, куда отправить email и пароль пользователя. Такое место называется API endpoint — адрес или функция, которая принимает данные. В этом случае она должна принять данные регистрации, создать аккаунт и вернуть понятный ответ: получилось или нет.

## 4. Деплой в production

До:

> Deploying now.

После:

> Это опубликует изменения в рабочую версию, которую могут увидеть реальные пользователи. Перед таким действием нужно подтверждение. Подтверждаешь деплой в production?
